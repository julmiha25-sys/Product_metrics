**Дашборд с RFM-анализом клиентской базы аптечной сети, реализованный на Metabase c использованием SQL (файлы в репозитории Metabase)**

**Проект «RFM-анализ клиентской базы аптечной сети»**

**_1.	Основные метрики_**
   
Для анализа клиентской базы были выбраны следующие классические метрики:

-частота совершения покупок Recency;
   
-количество покупок Frequency;

-сумма покупок Monetary.

Выбор метрик обоснован самим названием анализа «RFM-анализ».

**_2.	Основные гипотезы_**
   
Основные гипотезы при анализе клиентской базы аптечной сети:

1)В клиентской базе существуют как лояльные клиенты, так и «разовые». Поэтому клиентская база аптечной сети нуждается в сегментации для разработки таргетированных сценариев взаимодействия с каждой группой.
   
2)Определение списка ключевых клиентов, приносящих аптечной сети большую часть прибыли, необходимо  для активного развития и вовлечения их в дальнейшее взаимодействие, для максимального удержания. Здесь имеет смысл стимулировать клиентов нематериальными бонусами: приглашать на мероприятия, бесплатно присылать полезные экспертные материалы.

3)Определение списка «спящих» клиентов, совершивших покупки в прошлом, но не проявляющих активности в течение некоторого интервала времени, необходимо для их «активации». Здесь целесообразно предложить более дорогие продукты, а частоту рассылок можно увеличить, порекомендовать популярные товары,  действующие акции и  промокоды.

4)Определение списка клиентов, совершавших редко крупные покупки, необходимо для их мотивации. Здесь можно ограничиваемся стандартными рассылками или звонками.

5)Определение списка бесперспективных клиентов, совершавших давно и нечасто небольшие покупки, необходимо для того, чтобы не тратить на них большого количества ресурсов. Здесь целесообразно не отказываться от клиентов, а использовать низкобюджетные попытки активировать их с помощью бонусных баллов, промокодов или акций. 

6)Определение списка клиентов промежуточных категорий также позволит выявить стратегии для работы с ними (акция, звонок, опрос, рассылка и прочее).

**RFM-анализ клиентской базы аптечной сети**

Данный анализ подразумевает оценку клиентской базы по каждой метрике (RFM) по 3-х балльной шкале оценок (где 1 группа - лучшее значение, 3 группа – худшее).

**_Цель анализа – сегментация клиентской базы по метрикам и группам оценок._**

Отмечу, что для recency распределение на группы осуществлялось с использованием алгоритма адаптивной сегментации по локальным максимумам в диапазонах распределения. 

**_Достоинства этого метода:_** 

-учет реального распределения данных; 

-автоматическая адаптация к аномалиям (игнорирование пустых диапазонов, устойчивость к выбросам); 

-обнаружение естественных кластеров. 

Предварительный график recency подходит под характеристику данных, обрабатываемых этим методом.

Для frequency и monetary распределение на группы осуществлялось с помощью алгоритма сегментации по математическим 2-м пикам распределения (пик после впадины). Если данные за период попадут под унимодальное распределение, то деление диапазона осуществляется на квантили по 50% от первого пика.

**_Достоинства этого метода:_** 

-автоматическое определение типа распределения; 

-адаптивность к разным сценариям данных;

-минимизация внутригрупповой дисперсии;

-устойчивость к шумам.

Предварительные графики frequency и monetary подходят под характеристику данных, обрабатываемых этим методом.
